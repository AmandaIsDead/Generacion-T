# Trabajo Práctico - Mi primer AGENTS.md

## Proyecto
Proyecto de software de uso general. Este documento define las reglas que debe seguir un agente de IA al colaborar en el desarrollo.

## Comandos principales
- Leer la documentación antes de realizar cambios.
- Ejecutar las pruebas del proyecto cuando existan.
- Verificar que el código funcione correctamente antes de finalizar una tarea.

## Estilo de código
- Escribir código claro y ordenado.
- Utilizar nombres descriptivos para variables y funciones.
- Mantener una sangría consistente.
- Evitar duplicar código.
- Agregar comentarios solo cuando sean necesarios para mejorar la comprensión.

## Qué puede hacer el agente
- Corregir errores.
- Optimizar el rendimiento del código.
- Proponer nuevas funcionalidades.
- Refactorizar el código pero siempre sin cambiar su comportamiento.
- En caso de hacerlo, explicar los cambios realizados con comentarios.

## Qué NO debe hacer sin permiso
- Eliminar archivos o funcionalidades.
- Modificar la estructura principal del proyecto.
- Instalar nuevas dependencias.
- Cambiar configuraciones importantes.
- Realizar cambios que puedan afectar el funcionamiento del proyecto sin consultar primero.
